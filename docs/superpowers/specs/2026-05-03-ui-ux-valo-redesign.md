# VALOCUSTOM UI/UX & VALO要素強化 設計ドキュメント

## 目標
使いやすさの根本的改善とVALORANT公式UI風のビジュアル強化を両立する。

## アーキテクチャ方針
既存コンポーネントの構造を維持しつつ、以下のコンポーネントを修正・追加する。新規ファイルは最小限に抑える。

---

## 1. ステップバー（新規: `StepBar.tsx`）

### 概要
ページ上部に常時表示。現在のステップを赤くハイライト、完了済みはチェック付き。

### デザイン
- 4ステップ: `① プレイヤー追加 → ② モード選択 → ③ チーム結果 → ④ BANフェーズ`
- VALOのUI風に `clip-path: polygon(...)` で斜めカット形状を連結
- 現在: 赤背景 + 白文字
- 完了: 暗背景 + チェックマーク（✓）+ グレー文字
- 未到達: 暗背景 + グレー文字

### Props
```typescript
interface Props {
  current: 'input' | 'mode' | 'result' | 'ban';
}
```

## 2. プレイヤー入力の刷新（修正: `PlayerInputForm.tsx`）

### 入力フィールドの分割
- `名前` フィールドと `タグ` フィールドを**別々に分割**
- `#` を意識しなくて良くなる
- Tab キーで名前 → タグ → 追加ボタンに移動
- どちらかが空のとき追加ボタンは disabled

### まとめて追加ボタン
- 「まとめて追加」ボタンを追加
- 押すとテキストエリアが展開
- 改行区切りで複数人を一括追加（例: `Faker#JP1`）
- 各行をパースして順番に API 呼び出し
- 処理中は進捗表示（`2/5 取得中...`）
- 1人でもエラーがあれば、その行だけスキップしてエラー一覧を表示

### ローディング・エラー改善
- 取得中: プレイヤーカードの位置にスケルトンアニメーション（グレーのパルス）
- エラー: 赤いバナーで大きく・明確に表示（既存の小さいテキストから変更）

## 6. BANフェーズのリデザイン（修正: `BanPhase.tsx`）

### ロール別グループ表示
- valorant-api.com の `role.displayName` を使ってエージェントをロール別に分類
- タブ or セクションで「デュエリスト / コントローラー / センチネル / イニシエーター」に分ける

### BANエフェクト
- BANされたエージェントのアイコン上に赤い × マークオーバーレイ
- グレースケール + opacity 低下

### ターン表示強化
- 「チームA の BAN」「チームB の BAN」をチームカラー（青/赤）で大きく表示
- 残りBAN数をバッジで表示
