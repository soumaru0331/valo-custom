# VALORANT カスタムバランサー 設計仕様書

作成日: 2026-05-01

---

## 概要

VALORANTのカスタムマッチ用チーム分けWebアプリ。プレイヤーのゲーム名＋タグからRiot APIで自動的にランクとパフォーマンスデータを取得し、複数のモードで公平なチーム分けを行う。

---

## 技術スタック

| 項目 | 技術 |
|------|------|
| フレームワーク | Next.js (App Router) |
| スタイリング | Tailwind CSS + カスタムVALORANTテーマ |
| ホスティング | Render（常時稼働） |
| APIキー管理 | サーバー側 `.env` ファイル（クライアントに非公開） |
| 履歴保存 | ブラウザ localStorage |
| 外部API | Riot Games API |

## データ取得仕様

### 入力
- `ゲーム名#タグ`（例：`PlayerName#JP1`）
- 対応人数：2〜10人

### Riot API 呼び出しフロー
1. `Account API` → PUUID 取得  
   `GET /riot/account/v1/accounts/by-riot-id/{gameName}/{tagLine}`
2. `Match List API` → 直近コンペティティブ50試合のマッチID取得  
   `GET /val/match/v1/matchlists/by-puuid/{puuid}` （queue=competitive）
3. `Match Detail API` × 50 → 各試合の詳細取得  
   `GET /val/match/v1/matches/{matchId}`

### 取得・計算するデータ
| データ | 内容 |
|--------|------|
| 現在のランク | 最新コンペティティブ試合から取得 |
| パフォーマンススコア | KDA・HS率・勝率から独自計算（コンペ50試合） |
| よく使うエージェント | 上位3体（使用回数順） |
| アカウントレベル | サブ垢検知に使用 |

> **注意:** tracker.ggはAPIを非公開にしているためスクレイピング不可。「パフォーマンススコア」はRiotマッチデータから独自算出する。

### パフォーマンススコア計算式
各指標を0〜1に正規化してから加重平均を取る：
```
KDA正規化    = min(KDA平均 / 5.0, 1.0)   ← KDA 5.0を上限とする
HS率正規化   = HS率平均（0〜1の値）
勝率正規化   = 勝率平均（0〜1の値）

パフォーマンススコア = (KDA正規化 × 0.4) + (HS率正規化 × 0.3) + (勝率正規化 × 0.3)
※ 結果は 0.0〜1.0 の値
```

### サブ垢検知ロジック
以下の複数条件が重なった場合、プレイヤーカードに ⚠️ バッジを表示：
- アカウントレベル ≤ 30
- 直近50試合の勝率 ≥ 70%
- 直近50試合の平均KDA ≥ 3.0

## チーム分けロジック

### 総合スコア計算
```
総合スコア = ランク数値 × 0.7 + パフォーマンススコア(0〜1) × 0.3 × 40
※ ランク数値の最大が40（レディアント）なのでスケールを合わせる
```

### バランス計算
全プレイヤーの組み合わせを全探索し、2チームの合計スコア差が最小になる分割を選択。

### チーム分けモード（4種）

| モード | 説明 |
|--------|------|
| シンプル | 総合スコアで最も均等な2チームに分割 |
| ハンデあり | シンプル分割後、強いチームの上位プレイヤーにハンデを付与 |
| 前回被り少なめ（シンプル） | 前回同チームだったペアへペナルティを加算して分割 |
| 前回被り少なめ（ハンデあり） | 前回被り考慮 ＋ ハンデ付与 |

### 前回被り考慮ロジック
- localStorageから**直近1回分**のチームA・Bを読み込む
- 前回同チームだったペアが今回も同じチームになる場合、そのチーム分けの評価スコアに +5 ペナルティを加算して全探索で評価
- 複数のペアが被る場合は被ったペア数 × 5 のペナルティ

## ハンデシステム

### ハンデ付与条件
- チームの合計スコア差に応じて、強いチームの上位プレイヤーにランダムでハンデを1つ割り当て
- スコア差 5以上：上位1人にハンデ付与
- スコア差 10以上：上位2人にハンデ付与
- スコア差 15以上：上位3人にハンデ付与
- ハンデの☆レベルはスコア差に比例（5〜9→☆1、10〜14→☆2、15以上→☆3）
- 各プレイヤーに付与されるハンデは必ず1つ（同☆内からランダム選択）

### ハンデ一覧

#### ☆1（軽め）
- 京言葉で話す
- 韓国人風に話す
- 中国人風に話す
- 常に味方がどんなプレイしても褒める
- 報告を全部別ゲームで行う
- 英語で話す
- 毎ラウンド待機時間に食べたいご飯の話をする
- 死んだら早口で言い訳する
- 味方が1v1になったら歌い出す
- リロードするたびに一言コメントを入れる
- キルしたら必ず自分を褒める
- デスしたらゲーム環境のせいにする
- ミニマップを見た情報を実況風に話す
- 毎ラウンド開始時に目標を宣言する
- 味方の名前を毎回フルで呼ぶ
- 武器を買うたびに理由を説明する
- スキルを使う前に宣言する
- 敵を倒したら軽く謝る
- スパイク設置中にカウントダウンする

#### ☆2（中ハンデ）
- ジャンプしながら移動
- リロードは必ずカバー中に宣言してから
- ミニマップを見たら必ず共有しないといけない
- ラウンド中に一度は無意味なフェイク報告をする
- 武器は拾い物を使う
- キルした武器は次ラウンドも使う
- 1キルするまでしゃべれない
- 逆に常に何か喋り続ける
- 味方の指示に従う、指示待ち人間になろう
- 毎回違う武器を買う
- ガーディアンのみ
- ブルドッグのみ
- オーディンとアレスのみ購入可能

#### ☆3（重め）
- センシを半分もしくは2倍にする
- シェリフのみ
- スキル購入禁止
- 各ラウンドキルするまで走りのみ
- 毎ラウンド開始20秒は購入以外何もしない
- プラントされるまでローテ禁止
- デスしたら毎ラウンド全チャで理由を書く
- キルしてきた相手を褒めるコメントをする（毎回同じはだめ）
- スティンガーのみ
- オペレーターのみ
- サイドアームのみ

## BANフェーズ

チーム分け結果表示後に実施。

- チームA → チームB → チームA → … の順番で交互にBANを選択
- 1チームあたりのBAN数：設定可能（1〜3体、デフォルト1体）
- 全エージェントのアイコン一覧から選択するUI
- BANされたエージェントはグレーアウト表示
- BAN完了後、両チームのBAN結果を一覧表示してフェーズ終了

## ランダムマップ機能

現行VALORANTマップ一覧からランダムで1つを表示。結果画面のボタンから実行。

対象マップ（現行）：Abyss / Ascent / Bind / Breeze / Fracture / Haven / Icebox / Lotus / Pearl / Split / Sunset

## 履歴管理

- チーム分け確定時に localStorage へ保存
- 保存内容：チームA・チームBのプレイヤーリスト、日時
- 「前回被り少なめ」モード時に自動参照
- 手動でのリセット機能も提供

## 画面構成

### 1. プレイヤー入力画面
- ゲーム名#タグ入力フォーム（最大10個）
- 「取得」ボタンでRiot APIからデータ自動取得
- プレイヤーカード：ランクアイコン・エージェントアイコン3体・パフォーマンススコア・サブ垢疑惑バッジ

### 2. チーム分けモード選択画面
- 4モードをカード形式で選択

### 3. チーム分け結果画面
- チームA・チームBをカード形式で表示
- ハンデありモード：各プレイヤーのハンデを☆付きで表示
- 「ランダムマップ」ボタン
- 「もう一度」ボタン（再シャッフル）
- 「履歴に保存」ボタン

### 4. BANフェーズ画面
- エージェント一覧グリッド
- 各チームが順番にBANを選択
